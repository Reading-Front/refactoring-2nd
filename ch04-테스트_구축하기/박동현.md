## 코드 테스트 기본

### 경계값 테스트

```
ex) 요구사항
- 입력 A: 1~999 숫자 입력 가능
- 입력 B: 1~999 숫자 입력 가능
- 출력 C
- A * B = C

if(a > 0 && a < === 999) //올바름
else //잘못됨
```

### 상태 전이 테스트

상태를 모델화해서 테스트를 수행하는 방법입니다.
ex)
앱 기동하는 상태를 A
앱 기동하지 않는 상태를 B

상태 A에서 B로 변경하려면 앱 종료상태를 거쳐야 합니다.

시스템 가동 -> 입력대기 -> 메뉴 명령어 -> 대화 상자 열기 -> 대화상자 닫기 -> 종료
와같은 매트릭스 항목대로 작동하는가 확인이 필요합니다.

## 코드 기반 테스트

### 코드 기반 단위테스트

함수의 커버리지 비율을 측정해 로직의 확실성을 확인하는 화이트박스 테스트

```
ex)
- 프로그램 실행 도중, 시스템상의 이상 작동을 수행하지 않음(null pointer, 0으로 나누는 계산 등)
- 입력값과 그에 대응하는 기댓값을 출력함
- 모든 분기가 올바르게 처리됨(경곗값 테스트)
```

### 명령 커버리지(C0)

사실 명령 커버리지는 크게 의미있는 테스트라고 할 수 없습니다. 경곗값 테스트가 확실하게 수행되는지에 대한 여부를 알 수 없기 때문입니다.

```
//테스트 기준, 적어도 한 번은 프로그램의 모든 명령문을 실행한다.
if(con1 === 0){
	x = x + 1;
}

if(conf2 > 1){
  x = x * 2;
}
```

### 조건 커버리지(C1)

C0 커버리지의 문제를 해결하는 커버리지 기법으로 대부분의 경우에는 이 커버리지 방법으로 단위 테스트를 수행하는것이 좋음

조건 커버리지에는 각각의 판정 조건이 TRUE, FALSE인 결과를 적어도 1번씩 가지는 테스트 케이스를 작성해야합니다.

### 4자주 발생하는 단위 테스트의 오류

단순 커버리지만 신경쓰지말고, 함수의 in/out을 체크하는 것, 함수가 그 책임과 의무를 다 하는지 또한 중요하다.
